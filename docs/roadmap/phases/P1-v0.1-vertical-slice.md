# P1 — v0.1.0 Vertical Slice

## Objective

Deliver one reliable end-to-end focus session with a synchronized desktop companion, durable local record and coherent Roman pixel interface.

## In scope

- Mission and victory condition.
- Preset/custom duration.
- Authoritative session state service.
- SQLite migrations and repositories.
- Main/companion synchronization.
- Pause/resume/complete/abandon.
- Restart recovery.
- Basic local history and derived summary metrics.
- Show/hide and safe click-through companion control.
- Keyboard-operable custom window controls.
- Approved production assets for release-critical visual regions, or explicit written deferral.
- Unit, integration and critical-path E2E tests.

## Out of scope

Persistent campaign planning, automatic break doctrine, progression, ranks, analytics, AI and social features.

## Asset gates

Before production art integration:

- `PROD-COMP-001` must be supplied/approved or explicitly deferred.
- `PROD-UI-001` must be supplied/approved or explicitly deferred.
- Agents may not silently ship `scaffold-only` assets as final.

## Acceptance matrix

1. Given Idle, starting a valid mission creates exactly one active session.
2. Given Focusing, a second START is rejected.
3. Given sleep/throttling, displayed remaining time differs by less than one second after recovery.
4. Given app restart, the active session is recovered without duplicate records.
5. Given two completion commands, only one completion record exists.
6. Given companion click-through, the user can disable it from a global recovery path.
7. Given a removed monitor, the companion remains reachable.
8. All stored records survive app restart.
9. Essential actions and custom window controls are keyboard accessible.
10. Dynamic timer/task text remains live HTML and readable without decorative assets.
11. Production-required art has an approved registry status or a documented release waiver.
12. Five users can complete the defined test without facilitator correction.
