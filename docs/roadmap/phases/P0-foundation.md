# P0 — Foundation (`v0.0.x`)

## Objective

Remove product, visual, asset and architecture ambiguity before implementation agents multiply the codebase.

## Sub-releases

### v0.0.1 — Initial scaffold

- Product vision and MVP PRD.
- Canonical session model.
- Initial Tauri/React prototype.
- First generated companion assets.

### v0.0.2 — Pixel UI foundation

- Componentized Roman 16:9 dashboard.
- User-supplied reference breakdown.
- Generated 9-slice/texture scaffold assets.
- Mandatory user asset request gate.
- Asset registry.
- Coding standards and research source register.

### v0.0.3 — Interaction/visual validation

- Keyboard and control-state review.
- Deterministic visual-test harness.
- DPI/viewport presets.
- Production art request handoff.

### v0.0.4 — Architecture stabilization

- Cross-window authoritative state spike.
- SQLite migration/repository spike.
- Tauri capability separation.
- CSP/security decision.

## Exit criteria

- No unresolved contradiction in session states.
- Every v0.1 task has acceptance criteria and asset dependencies.
- Companion interaction can be demonstrated.
- Main dashboard functions without relying on a flattened background image.
- Missing production assets are explicit `request-user` registry entries.
- TypeScript/React, Rust/Tauri, CSS, testing and security standards are accepted.
- Cross-window ownership and persistence choices are recorded in ADRs.
