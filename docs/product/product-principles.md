# Product Principles

1. **One mission is stronger than many visible options.**
2. **The companion informs; it does not demand attention.**
3. **A missed session is information, not moral failure.**
4. **Rest belongs inside the doctrine.**
5. **Real work creates progression; progression never replaces real work.**
6. **Local-first is the default trust posture.**
7. **Roman language supports clarity instead of obscuring it.**
8. **Every feature must improve activation, focus continuity or closure.**
