# Starter Manifest — v0.0.2

## Executable scaffold

- React/Vite application.
- Tauri main and companion windows.
- Custom main-window command controls.
- Timestamp timer prototype.
- Componentized Roman campaign dashboard.
- Generated 9-slice frames, textures, icons and sprite sheets.
- Unit/component tests, Playwright E2E scaffold, deterministic preview capture, lint and version synchronization configuration.

## Product and UX package

- Vision, principles, personas and MVP PRD.
- Supplied-reference breakdown.
- Main dashboard and companion behavior.
- Pixel visual system and production replacement plan.
- Accessibility and interaction requirements.

## Agentic execution package

- Authoritative `AGENTS.md` and root-level `USER-INPUT-GATE.md`.
- Mandatory asset stop/request protocol.
- Asset registry, checksum manifest, mandatory request template and generated-file standards.
- Task prompt format and workflow.
- Claude and Copilot repository instructions.

## Engineering package

- Domain model, state machine and events.
- Architecture, persistence plan and ADRs.
- TypeScript/React, FSD boundaries, CSS, Rust/Tauri, testing, security and versioning standards.
- Official technical research source register.
- Phased roadmap and asset-aware backlog.
- QA strategy, edge cases and usability script.

## Deliberately incomplete

- authoritative cross-window state ownership,
- SQLite implementation,
- CSP hardening,
- tray, shortcut, notifications and autostart,
- approved production UI/character art,
- persistent campaign and progression domains,
- installer signing and updater.

These are explicit roadmap items, not accidental omissions.
